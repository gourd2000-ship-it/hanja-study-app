# 한자마루 (Hanja Maru) - 제품 설계 기록 (PDR)

## 1. 프로젝트 개요
'한자마루'는 사용자가 한자 급수 시험(8급 ~ 6급)을 효율적으로 준비할 수 있도록 돕는 웹 기반 학습 애플리케이션입니다. 망각 곡선 이론에 기반한 복습 알고리즘과 테마별 한자 배치를 통해 학습 효과를 극대화합니다.

## 2. 주요 기능
### 2.1. 급수별 학습 커리큘럼
*   **제공 급수**: 8급, 7급A/B, 6급A/B/C (총 300자)
*   **테마별 배치 (6급)**: 가나다순이 아닌 의미 단위(동작, 자연, 사회 등)로 한자를 재배치하여 암기 효율 향상
*   **학습 스케줄**: 20일 완성 코스로 구성 (신규 학습 + 누적 복습)

### 2.2. 학습 및 퀴즈 시스템
*   **오늘의 한자**: 스케줄에 맞춰 매일 새로운 한자와 복습 한자 제공
*   **퀴즈**: 4지선다형 퀴즈를 통해 학습 내용 점검
*   **오답 관리**: 틀린 한자를 자동으로 '약점 한자'로 등록하여 집중 학습 지원

### 2.3. 진행 상황 관리
*   **로컬 데이터 저장**: `localStorage`를 사용하여 별도의 로그인 없이 브라우저에 학습 진도 저장
*   **마이그레이션**: 앱 업데이트 시 기존 데이터를 보존하며 새로운 급수 데이터를 자동으로 추가

## 3. 학습 알고리즘 (누적 복습 시스템)
'한자마루'는 에빙하우스의 망각 곡선 이론을 바탕으로, 한 번 배운 내용을 잊어버리기 전 적절한 시점에 다시 노출시키는 **누적 복습(Cumulative Review)** 시스템을 핵심으로 합니다.

### 3.1. 학습 구성의 기본 원리
매일의 학습은 **'오늘의 신출 한자'**와 **'과거 학습 내용의 재노출'**이 조합되어 이루어집니다. 학습이 진행될수록 복습해야 할 한자가 누적되어 기억의 유효 기간을 늘립니다.

### 3.2. 누적 복습 진행 단계 및 구체적 예시
사용자는 신규 한자 학습 후 일정 주기에 따라 반복 노출을 경험합니다. (시스템 내부적으로는 D+2, D+4, D+7 등의 오프셋을 활용하여 복습 대상을 선정합니다.)

#### [예시 1] 2차 누적 복습 진행 흐름 (D+2 중심)
초기 학습 단계에서 신규 한자가 어떻게 첫 번째 복습으로 이어지는지 보여줍니다.
*   **1일차**: 신출 4개 (복습 없음)
*   **2일차**: 신출 4개 (복습 없음)
*   **3일차**: **신출 3개** + **1일차 복습 4개** (1차 누적복습, 총 7개)
*   **4일차**: **신출 3개** + **2일차 복습 4개** (1차 누적복습, 총 7개)
*   **5일차**: **신출 3개** + **3일차 복습 3개** (1차 누적복습, 총 6개)
*   **6일차**: **신출 3개** + **4일차 복습 3개** (1차 누적복습, 총 6개)

#### [예시 2] 3차 누적 복습 및 재복습의 결합 (2차 누적복습 중심)
학습 중반부로 갈수록 이전의 복습 내용이 다시 한번 노출되어 장기 기억으로 전환됩니다.
*   **7일차**: **신출 3개** + **5일차 복습 3개** + **1일차 재복습 4개** (1차+2차 누적복습, 총10개)
*   **8일차**: **신출 3개** + **6일차 복습 3개** + **2일차 재복습 4개** (1차+2차 누적복습, 총10개)
*   **9일차**: **신출 3개** + **7일차 복습 3개** + **3일차 재복습 3개** (1차+2차 누적복습, 총9개)
*   **10일차**: **신출 3개** + **8일차 복습 3개** + **4일차 재복습 3개** (1차+2차 누적복습, 총9개)
이처럼 '한자마루'는 단순 암기가 아닌, 과학적으로 설계된 스케줄에 따라 자연스럽게 반복 학습이 이루어지도록 설계되었습니다.

### 3.3. 11일차 이후 및 심화 복습기 (Reinforcement)
학습 중반 이후부터는 신규 단어 학습의 마무리와 장기 기억화를 위한 집중 복습이 이루어집니다.

1.  **11일~15일 (학습 마무리 단계)**: 매일 3개씩 새로운 한자를 학습하며, 15일차에 해당 급수의 **총 50자 학습이 완료**됩니다. 이 기간은 가장 많은 복습 한자가 노출되는 '누적 복습의 절정' 구간입니다.
2.  **16일~20일 (심화 복습기)**: 15일간 배운 50자를 완벽하게 다지는 기간입니다.
    *   **신규 단어**: 0개 (추가 학습 없음)
    *   **복습 구성**: 매일 20~30개의 한자를 무작위 및 주기적으로 노출하여 전체 50자를 장기 기억으로 전이시킵니다.

### 3.4. 약점 보완 시스템
*   퀴즈에서 틀린 한자는 자동으로 `weakHanjaIds` 목록에 저장됩니다.
*   학습 화면 하단에 '약점 한자' 섹션을 별도로 제공하여, 사용자가 언제든 부족한 부분을 추가 학습할 수 있도록 유도합니다.

## 4. 기술 스택
*   **Frontend**: React, TypeScript, Vite
*   **Styling**: Vanilla CSS / Tailwind CSS
*   **State Management**: Custom React Hook (`useStudy`)
*   **Testing**: Vitest, React Testing Library
*   **Deployment**: Vercel

## 4. 아키텍처
*   **Data Layer (`src/data/hanja.ts`)**: 전체 한자 데이터 및 인터페이스 정의
*   **Logic Layer (`src/hooks/useStudy.ts`)**: 학습 스케줄 계산, 진도 저장, 퀴즈 로직 등 핵심 비즈니스 로직 담당
*   **UI Layer (`src/App.tsx`)**: 사용자 인터페이스 렌더링 및 인터랙션 처리

## 5. 향후 과제 (Future Roadmap)
### 5.1. 기능 확장
*   **고급 급수 추가**: 5급 이상의 한자 데이터 및 커리큘럼 확장
*   **멀티미디어 학습**: 한자 획순 애니메이션 및 원어민 발음 기능 추가
*   **다양한 퀴즈 모드**: 필기 인식 퀴즈, 매칭 게임 등 흥미 유발 요소 추가

### 5.2. 기술 개선
*   **클라우드 동기화**: Firebase 또는 Supabase를 연동하여 기기간 학습 데이터 동기화
*   **PWA 지원**: 오프라인에서도 학습이 가능하도록 Progressive Web App 도입
*   **통계 대시보드**: 학습 시간, 정답률 추이 등을 시각화하여 제공

### 5.3. 사용자 경험 (UX)
*   **게이미피케이션**: 연속 학습 일수(Streak), 배지 시스템 도입으로 동기 부여
*   **다크 모드 지원**: 사용자 선호에 따른 테마 선택 기능
